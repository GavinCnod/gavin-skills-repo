# GitHub Agent Skills 仓库构建计划 (Revised)

根据你的要求，我参照了 `anthropics/skills` 的官方结构，并设计了中英双语的 README 方案。这个结构通用性强，不局限于特定的编程语言，非常适合作为个人技能库 (IP) 的展示。

## 1. 仓库结构设计 (Repository Structure)

我们将采用标准的 Agent Skills 结构，核心在于每个 Skill 都是一个独立的文件夹，并包含 `SKILL.md` 定义文件。

```text
gavin-skills-repo/
├── README.md           # [英文/主页] 默认英文，顶部包含跳转中文的链接
├── README_zh.md        # [中文说明] 完整的中文介绍
├── LICENSE             # MIT License (最宽松的开源协议，适合个人 IP 传播)
├── .gitignore          # 通用忽略配置 (DS_Store, .env, __pycache__, node_modules 等)
└── skills/             # [核心目录] 存放所有技能
    ├── _template/      # [标准模板] 包含标准的 SKILL.md 结构
    │   └── SKILL.md    # 定义技能元数据 (name, description) 和指令
    └── example-skill/  # [示例技能] 用于展示如何编写一个 Skill
        ├── SKILL.md
        └── scripts/    # (可选) 如果技能需要脚本，放在这里
```

## 2. 核心文件内容规划

### 2.1 README.md (English - Default)
- **Header**: Project Title & "Created by Gavin".
- **Language Switch**: `[🇨🇳 中文说明](README_zh.md)` (显著位置).
- **Introduction**: 简述仓库目的（Personal Agent Skills Library）。
- **Skills Showcase**: 表格形式列出技能，包含 `Name`, `Description`, `Path`.
- **Usage**: 解释如何加载或使用这些 Skill（引用 `SKILL.md` 标准）。
- **Author / About Me**: 你的个人介绍板块，链接到你的其他主页。

### 2.2 README_zh.md (中文版)
- **内容**: 与英文版一一对应，但使用中文描述，方便国内用户或中文语境下的交流。

### 2.3 skills/_template/SKILL.md (核心规范)
参照 Anthropic 标准，建立统一的 Skill 定义文件：
```markdown
---
name: my-new-skill
description: 一句话描述这个技能的作用
---

# My New Skill

[详细的指令说明...]

## Examples
- 示例 1...

## Guidelines
- 注意事项...
```

## 3. 执行步骤 (Implementation Steps)

1.  **创建目录结构**: 建立 `skills/_template` 和 `skills/example-skill`。
2.  **生成基础文件**: 编写通用的 `.gitignore` 和 `LICENSE` (MIT)。
3.  **编写双语文档**:
    - 先写 `README.md` (英文)。
    - 再写 `README_zh.md` (中文)。
4.  **创建模板**: 在 `skills/_template/SKILL.md` 中写入标准的前置元数据 (Frontmatter) 格式。
